Background

Most academic libraries in the United States organize their books using a system invented for the Library of Congress. For this assignment, you will define a Book class that can keep track of a book’s title, author, and call number, and can properly determine the order of two books based on their call numbers.

Elements of a call number

Here are some examples of Library of Congress call numbers:

QA1133 .D53 1999
QA276.15 .H35 1998
Q2044.J3 C43 2000

A call number starts out with one to three letters representing a class of subjects. Following the letter or letters is a number that specifies a subject within that class. The number can range from 1 to 9999 and may or may not include a decimal component.

After the initial letters and numbers comes a period, a letter, and one or more numbers. This is called the cutter number (it’s named after American librarian Charles Ammi Cutter). The cutter number may represent one of a number of things: a more specific topic, the author’s last name, or the first word in the title of the book. Ultimately, it just provides a shortcut so a librarian can shelve the book in the right place without having to remember if they’re supposed to go off the title, the author, the topic, etc.

Some books have two cutter numbers. The third example above is a "double-cuttered" call number. The first cutter number in that example, J3, specifies the topic; the second cutter number, C43, is based on the author’s name.

Most Library of Congress call numbers end with the year the work was printed, represented by four digits.

Magic methods

Python allows classes to define how they interact with various features of the language, including how they should be represented as strings and how they should behave with respect to various operators, among other things. The way a class defines these behaviors is by defining special methods whose names start and end with double underscores. You have already seen __init__() and __str__(). For this assignment, we’ll introduce two additional magic methods, __repr__() and __lt__(). If you’d like to read about Python’s other magic methods, you can check out the official documentation or this tutorial.

__repr__()

The __repr__() method should return a "formal" string representation of an object. If possible, this string representation should be a valid Python command that could be used to re-create the object. It requires one parameter, self.

The __repr__() method works with the built-in repr() function: if object a has a __repr__() method, you can get the formal string representation of the object by using repr(a).

More details...

Imagine you have a class called Dog that takes one required argument, a string indicating the dog’s name. Imagine that the class stores this value in an attribute called name. The __repr__() method of this class might look like this:

    def __repr__(self):
        return f"Dog({repr(self.name)})"

Notice that the __repr__() method itself calls the repr() function. That’s because self.name is a string, and strings need quotes around them, and the choice of whether to use single or double quotes can depend on what’s inside the string. Strings can also contain special characters that need to be escaped. When you pass a string to repr(), it makes sure to use the right quotes and to escape any special characters so that if you copied its result and pasted it into a Python script, it would evaluate to the exact string you passed to repr(). Take a look at the following examples to see how this affects the representation of various instances of our Dog class:

>>> gd1 = Dog("Fido")
>>> print(repr(d1))
Dog('Fido')
>>> d2 = Dog("Billy's puppy")
>>> print(repr(d2))
Dog("Billy's puppy")
>>> d3 = Dog("""This dog's name
... contains a line break""")
>>> print(repr(d3))
Dog("This dog's name\ncontains a line break")

"Fido" gets single quotes because it contains no special characters. "Billy’s puppy" gets double quotes because it contains an apostrophe (e.g., a single quote). And the third dog’s admittedly absurd name gets double quotes and a \n to represent the newline. By using repr() inside our __repr__() method, the name automatically gets represented as a valid Python string literal, no matter what tricky characters it may contain.

__lt__()

The __lt__() method defines how an object behaves when it occurs to the left of the < operator. It requires two parameters, self and other, where other is a value that occurs to the right of the < operator. If your code contained the expression a < b, the __lt__() method of a would get invoked, so a would be the value of self and b would be the value of other. The __lt__() method should return True if self should be treated as less than other, otherwise False.

The __lt__() method is also the method used by sort() and sorted() to compare values. This is why we are interested in this method for this assignment.

More details...

Imagine a class called Rectangle with attributes width and height and an area() method that returns the product of these two attributes. We might decide that rectangles with a smaller area should be considered less than rectangles with a larger area. Here’s how we might implement the __lt__() method in this case:

    def __lt__(self, other):
        return self.area() < other.area()

If we wanted to make instances of our class responsive to the full set of comparison operators (>, ==, ⇐, >=), we would also have to define methods __gt__(), __eq__(), __le__(), __ge__(). (We can also define __ne__() to make the != operator work, but by default it calls __eq__() and negates the result. You’d think the other operators would have similar smart default behaviors, but for whatever reason, they don’t.)

Instructions

Using the provided template, create a Book class and a read_books() function as described below.

Book class

Create a class called Book with the following methods:

__init__()

Define an __init__() method with three parameters:

  • self

  • callnum: the call number for the book (this will be a string)

  • title: the title of the book (also a string)

  • author: the author of the book (also a string, possibly empty)

Your __init__() method should define attributes callnum, title, and author corresponding to the parameters above.

__lt__()

Define a __lt__() method with two parameters:

  • self

  • other: another instance of the Book class

This method should return the boolean value True if self.callnum sorts before other.callnum. Otherwise, it should return the boolean value False. The rules for sorting call numbers can be found at https://libguides.colostate.edu/howtodo/findingbooks#s-lg-box-wrapper-19802423.

Hints:

You may wish to write a separate method to parse the call number into its various parts. I highly recommend using regular expressions to parse the call numbers. Some elements of a call number are optional. Note that a question mark after a group of parentheses tells Python to match zero or one instance of the sub-expression in parentheses. Note also that you can have sub-expressions within sub-expressions.

One way to sort two values according to multiple criteria is to create a tuple for each value where the first item in each tuple is the highest-priority criterion, the second item is the second-highest criterion, etc. Alternatively, you can use a conditional statement.

When comparing call numbers, watch your data types. The first set of numbers should be treated like regular numbers (e.g., 999 comes before 1000; 815.5 comes before 815.6; etc.). Other numbers in a call number are treated like "decimals" (e.g., 1000 comes before 999). Numbers that are treated like "decimals" can also be treated like strings, since '1000' comes before '999' using string sorting rules.

__repr___()

Define a __repr___() method with one parameter, self. __repr__() should return a string that could be used to re-create the current instance of Book. For example, if you create a Book object with call number A101 .B23 2020, title "Belinda’s Guide to Sprockets", and author Belinda Buxton, that object’s __repr___() method should return the following as a string:

Book('A101 .B23 2020', "Belinda's Guide to Sprockets", 'Belinda Buxton')

read_books()

Outside of the class, define a function called read_books(). It should have a single parameter, a path to a file containing information about books. The file will be a text file in UTF-8 encoding, and each line in the file will contain a book title, a tab character, the author of the book, another tab character, and the book’s call number at the Library of Congress.

This function should open the file for reading and build a list of instances of your Book class, one per line in the file. The function should return the list once it is complete.

Please note: the columns in the file are defined in a different order than the parameters in Book.init().

Docstrings

Please write a docstring for the Book class that describes the purpose of the class and the data type and purpose of each attribute of the class.

Write a docstring for each method and function above with the exception of __init__(). Each docstring should describe the purpose of the function or method, the data types and purposes of each parameter, the data type and purpose of the return value (if any), and any side effects caused by the function or method.
